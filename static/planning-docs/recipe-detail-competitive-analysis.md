# Competitive analysis — dev-tool doc patterns for the recipe detail frame

Goal: readability + AI-readiness, as an EVOLUTION of today's page (not a jarring change).
The recipe page is shaped like an API/SDK reference: identifier + description + options (params) +
examples (before/after) + usage (command) + data tables. So API-reference patterns transfer well.

## What the leaders do

### GitHub REST API docs (cited by Jayd)
* Broke long pages into categories/subcategories; operations moved into the sidebar (perf + scannability).
* **Three-column layout**: nav | content | examples. Parameter tables redesigned to scan in a smaller width.
* **Sticky example column** so you can cross-reference params ↔ example without scrolling.
* Source: https://github.blog/developer-skills/github/improved-rest-api-documentation/

### Stripe / Twilio API reference (the "gold standard" two-panel)
* Two-panel: prose left, **sticky code/example panel right**. Clean and uncluttered despite high density.
* Organized around developer GOALS ("Send a message"), not just a list of endpoints.
* Twilio ships a **"Copy as Markdown"** page action.
* Sources: https://docs.stripe.com/api , https://www.twilio.com/docs ,
  https://apidog.com/blog/twilios-api-the-other-gold-standard-and-why-its-stripes-true-equal/

### Amplitude docs (cited by Jayd)
* Recently consolidated multiple sites into one Developer Center; fully open-source docs.
* Consistent internal structure per section; getting-started-first; centralized home.
* Source: https://amplitude.com/docs , https://amplitude.com/blog/developer-experience-week

## AI-readiness patterns (the emerging standard, 2025–26)
* **"Copy page" dropdown** = the headline pattern: Copy as Markdown · copy shareable `.md` link ·
  **Open in ChatGPT** · **Open in Claude**. (Fern, Mintlify, GitBook, DatoCMS.)
* **`llms.txt` / `llms-full.txt`**: a site-level index of every page in markdown for LLMs. 600+ sites by
  mid-2025 incl. Stripe, Anthropic, Cursor, Perplexity, Cloudflare. Auto-generated by GitBook/Fern/Mintlify.
* **Per-page `.md` endpoint** (content negotiation / `.md` URL) so AI tools fetch clean markdown.
* Markdown is ~80% fewer tokens than rendered HTML (Cloudflare) → cheaper/faster for AI tools.
* Sources: https://www.mintlify.com/blog/simplifying-docs-with-llms-txt ,
  https://buildwithfern.com/learn/docs/ai-features/llms-txt ,
  https://posthog.com/docs/ai-engineering/markdown-llms-txt ,
  https://www.datocms.com/product-updates/llm-ready-documentation-export-any-page-as-markdown

## Transferable patterns, ranked by fit-vs-disruption for OUR page

| Pattern | Readability | AI-ready | Disruption | v1 frame? |
| --- | --- | --- | --- | --- |
| "Copy page" header action (Copy as MD / Open in ChatGPT+Claude) | – | High | Low | YES |
| Metadata/action header band (group name, source, license, CTA) | High | Med | Low | YES |
| Elevate primary CTA ("Try in Platform") near top | – | – | Low | YES |
| Typographic hierarchy on description + section headers | High | Low | Low | YES |
| Examples elevated / lightly emphasized | High | – | Low | YES |
| Sticky right example panel (full Stripe/GitHub two-panel) | High | – | HIGH (reflow) | NO — too jarring; defer |
| `llms.txt` / per-page `.md` endpoint | – | High | site-level | Phase 2 (FE/build) |

## Recommended direction (evolution, not revolution)
Keep today's single-column body and section ORDER untouched. Layer in:
1. A **page header zone**: title + compact metadata row (identifier, license, source links) +
   **primary CTA "Try in Platform"** + an **AI "Copy page" menu** (Copy as Markdown, Open in ChatGPT, Open in Claude).
2. **Readability polish**: stronger heading hierarchy, de-emphasize the long italic description blob into clean
   lead prose, tidy the options/data tables for scanning.
3. Leave examples / options / definition / data tables where they are, lightly restyled.

Explicitly NOT doing in v1: the full two-panel sticky layout (biggest reflow risk), and llms.txt/.md endpoints
(site/build-level, phase 2). Everything reuses existing Neo components.
